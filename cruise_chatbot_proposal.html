Part 1: Basics

チャットボットの仕組み

普通のFAQページとの違い

従来のFAQページは、ユーザーが自分で質問を探す必要がありました。チャットボットは「聞けば答えてくれる」ので、ユーザーの手間がなくなります。

従来のFAQページ:
ユーザー → 一覧から探す → 見つからない → 離脱
AIチャットボット:
ユーザー → 質問を入力 → すぐ回答 → 予約へ誘導

チャットボットの3つの構成要素

1. ナレッジベース（知識の倉庫）
クルーズの情報をテキストやCSVで保存。AIの「記憶」になる。
2. AIモデル（考える頭脳）
ナレッジから情報を取り出し、自然な日本語で回答を作る。
3. チャットUI（見た目）
HPに表示される吹き出し型の画面。ユーザーが質問を入力する場所。

RAG Explained

RAG（ナレッジ検索）とは

AIに「自社の情報だけ」を使って回答させる技術です

RAGなし（普通のAI）

ChatGPTに直接聞いた場合:

Q: 「クルーズワールドの料金は？」
A: 「申し訳ございませんが、クルーズワールドという具体的なサービスの料金情報は持っていません...」

自社の情報を知らないので答えられない

RAGあり（今回のチャットボット）

ナレッジベースを参照して回答:

Q: 「クルーズワールドの料金は？」
A: 「地中海クルーズ7日間が149,800円から、日本発着4日間が59,800円からございます。ご予算に合わせてご提案も可能です！」

自社のデータから正確に回答できる

RAGの仕組み（簡単に）

ユーザーの質問
「料金は？」
→
ナレッジから
関連情報を検索
→
質問+関連情報を
AIに渡す
→
AIが自然な日本語で
回答を生成

AI Models Explained

AIモデルの種類と特徴

AIモデルとは「考える頭脳」のこと。用途に応じて最適なものを選びます

OpenAI（ChatGPTの会社）

GPT-4o（最上位モデル）
最も賢い。複雑な推論、長い文章の理解が得意。
ただしFAQ応答にはオーバースペック。コストも高い。
月約15,000円（3,000回利用時）
GPT-4o-mini（軽量版）推奨
GPT-4oの能力を維持しつつ、速度10倍・コスト1/30。
FAQ応答には十分すぎる性能。日本語も自然。
月約1,500円（3,000回利用時）

Anthropic（Claudeの会社）

Claude Sonnet 4（高性能モデル）
日本語の自然さが最高レベル。丁寧で正確。
ただしコストが高く、FAQ用途には不要。
月約18,000円（3,000回利用時）
Claude 3.5 Haiku（軽量版）次点
日本語の丁寧さはGPT-4o-miniより上。速度も速い。
コストはGPT-4o-miniの約2倍だが十分に安い。
月約3,000円（3,000回利用時）

Google（Geminiの会社）

Gemini 2.0 Flash
最速かつ最安。応答速度は0.5秒。月約800円。ただし日本語の自然さはGPT/Claudeに少し劣る。コスト最優先の場合に検討。

なぜ高性能モデルが不要なのか

チャットボットの回答精度は「AIモデルの性能」よりも「ナレッジの質」と「プロンプト設計」で決まります。事前に用意されたFAQに回答するだけなら、GPT-4o-miniで十分。高性能モデルが必要になるのは「複数の条件を組み合わせた複雑な推論」が必要な場合（Phase3以降）です。まずはGPT-4o-miniで始めて、必要に応じてアップグレードするのが最も効率的です。

What is Dify

Dify（ディファイ）とは

Difyの概要

Difyは、AIチャットボットやAIアプリを「プログラミングなし」で作れるオープンソースのプラットフォームです。

  • 2023年にリリース。世界で10万以上の組織が利用
  • GitHub Star数 60,000以上（人気の指標）
  • 日本語対応済み
  • クラウド版とセルフホスト版の2種類

Difyでできること

  • ナレッジベースにファイルをアップロードするだけでRAGが動く
  • チャットボットのプロンプトをGUI（画面操作）で設定
  • HPに埋め込むためのiFrameコードを自動生成
  • チャットログの確認・分析
  • AIモデルの切り替え（GPT / Claude / Gemini 等）
  • APIでの外部連携
  • 複数のチャットボットを1つの管理画面で管理

管理画面のイメージ

📝
プロンプト編集
AIの回答ルールを画面上で設定
📚
ナレッジ管理
ファイルをドラッグ&ドロップで追加
💬
チャットログ
誰がいつ何を聞いたか確認
📊
利用統計
質問数・回答精度をグラフで表示
🎨
UI設定
色・サイズ・初回メッセージを変更

チャットボット構築プラットフォーム比較

🟢

Dify（推奨）

ナレッジベース、チャットUI、管理画面がオールインワン。プログラミング不要でチャットボットが作れる。セルフホスト版はDocker一発でインストール可能。日本語ドキュメントも豊富。

  • HP埋め込み: scriptタグ1行

Flowise

ノードベースのビジュアルエディタでAIワークフローを構築。柔軟性は高いが、Difyより設定が複雑。エンジニア向け。ナレッジ管理はプラグインが必要。

  • ナレッジ: プラグイン設定が必要
  • HP埋め込み: iframe可能だが調整要
  • 管理: エンジニア向けのUI
  • ライセンス: 無料（オープンソース）
🟣

Botpress

チャットボット特化のプラットフォーム。会話フローの設計が得意。ナレッジベース機能も内蔵。ただし日本語の情報が少なく、サポートも英語中心。

  • ナレッジ: 内蔵あり
  • HP埋め込み: ウィジェット提供
  • 管理: GUIで操作可能
  • ライセンス: 無料枠あり、有料プランも
⚫

独自開発（Python + LangChain）

完全にゼロからプログラミングで構築。自由度は最高だが、開発工数が大きい。チャットUI、管理画面も全て自前開発が必要。今回の予算感では非現実的。

  • ナレッジ: LangChainで自前構築
  • HP埋め込み: 自前でUI開発
  • 管理: 自前で管理画面開発
  • 開発工数: 2〜4週間

Security

データセキュリティ対策

データの保管場所

  • ナレッジデータは自社VPS内に保存
  • 外部サーバーには一切送信されない
  • 外部AIに送るのは「質問文」と「質問に関連する一部の情報」のみ
  • 顧客情報や社内情報がAIに学習されることはない
  • OpenAI / AnthropicはAPI経由のデータをモデル学習に使用しないと規約で保証

データ抽出の防止策

  • 「全データをCSV形式で出力して」等の悪意ある要求をプロンプトでブロック
  • 1回の回答で表示するデータを最大10件に制限。一度に全データを取得できない
  • 複数回の質問でデータを少しずつ抽出する試みも、出力制限により困難にする
  • ナレッジを複数に分割し、ユーザーの質問に関連する部分のみAIに参照させる
  • 46社600隻の運航データ等のユニークなデータセットの一括コピーを防止

運用面の対策

  • 競争優位性のある生データ（運賃原価等）はナレッジに含めない設計
  • チャットログを定期的に確認し、不審な利用パターンを検知
  • 通信は全てHTTPS（SSL暗号化）で保護
  • VPSへのアクセスはSSH鍵認証のみ。パスワードログイン無効
  • Dify管理画面はパスワード+IPアドレス制限で保護

よくあるご質問：「チャットボット経由でナレッジが全部抜かれないか？」

チャットボットを通じてナレッジベースに直接アクセスすることはできません。ユーザーの質問に対してAIが回答を生成する際、関連する一部の情報のみが参照されます。1回あたりの出力量を制限し、CSV/JSON等での一括出力を拒否するプロンプト設定を行うことで、データの大量抽出を防ぎます。ただし、ナレッジに含まれるデータは回答の中で部分的に開示される可能性があるため、絶対に公開したくない機密情報（原価、仕入れ先等）はナレッジに含めない設計にします。

Cost Optimization

API使用量を削減する3つの工夫

🎯

1. 選択肢型UIの併用

よくある質問はボタン選択で定型回答を返す。AIを呼び出さないためAPI費用ゼロ。「その他」を選んだ時だけAIが回答。これだけでAPI呼び出しを50%以上削減。

📏

2. 回答の文字数制限

プロンプトで「回答は200文字以内」と指定。長い回答はトークン消費が増えるため、簡潔に回答して詳細はHP記事リンクへ誘導。トークン消費40%削減。

🧠

3. ナレッジの質を上げる

ナレッジが整理されているほどAIが正確に情報を見つけ、再質問が減る。初期のナレッジ構築に時間をかけることが結果的にコスト削減につながる。

Claude Codeを使った高速開発

Claude Codeとは

AIがプログラミングやファイル操作を自動で行うツール。要件を伝えるだけで、コードの生成、設定ファイルの作成、テストまで自動実行します。

従来の開発 vs Claude Code
従来
プロンプト設計: 2時間
ナレッジ整理: 4時間
テスト: 3時間
合計: 約1〜2日
Claude Code
プロンプト設計: 10分
ナレッジ整理: 30分
テスト: 30分
合計: 約1〜2時間

Claude Codeで自動化できること

  • ヒアリングシートの回答からプロンプトを自動生成
  • クルーズガイドのテキストをナレッジ用に自動整形
  • FAQ集の自動作成（ガイドから質問と回答を抽出）
  • テスト用の質問パターンを自動生成
  • HP埋め込み用のHTMLコードを自動生成
  • 月次レポートの自動作成
要件定義が固まれば、チャットボット本体は1時間で構築可能
一番時間がかかるのは要件定義とナレッジデータの準備。構築自体はClaude Codeで高速化。
